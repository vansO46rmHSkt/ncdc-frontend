## 概要

NCDC株式会社様における採用選考課題の提出物になります。

## 利用技術

原則として、以下の方針で選定しています。

1. 広く利用されており信頼性が高いこと
2. 利用経験が長いこと
3. 軽量で簡潔に利用できること

| 用途           | 名称               | 理由                                                   |
| -------------- | ------------------ | ------------------------------------------------------ |
| フロントエンド | React / TypeScript | 利用経験が長いため                                     |
| 開発環境       | Vite / Vitest      | ビルドやテストが高速なため                             |
| ルーティング   | React Router       | 標準的なクライアントサイドルーティングのため           |
| データ取得     | SWR                | 再検証により、複数コンポーネント間の状態を同期するため |
| スタイリング   | Tailwind CSS       | 簡潔な記述でスタイルを定義できるため                   |

## 設計方針

- カスタムフックを利用し、ビジネスロジックとUIを分離
- 共通して利用するコンポーネントは共通コンポーネントとして分離
- SWRを利用し、サイドメニューとメインページの状態を同期
- カスタムフックに対して自動テストを実装し、ビジネスロジックの正しさを検証

### フォルダ構成

```
├── app
│   ├── App.tsx
│   └── routes
│       └── content             # /contentで表示されるページのコンポーネント
│           ├── Main.tsx
│           ├── SideMenu.tsx
│           └── root.tsx
├── assets                      # 画像など静的なファイルを含むフォルダ
├── components                  # アプリケーション全体で共通して利用するコンポーネントを含むフォルダ
│   └── ui
│       ├── button              # ボタン系のコンポーネント
│       ├── input               # 入力系のコンポーネント
│       └── loading             # ローディングで利用するコンポーネント
├── features                    # 機能単位のコンポーネントやカスタムフックを含むフォルダ
│   └── content
│       ├── components
│       └── hooks
│           └── __tests__
└── lib                         # アプリケーション全体で利用する関数などを含むフォルダ
```

## セットアップ手順

### 手順

1. 依存関係のインストール

   ```bash
   npm install
   ```

2. 環境変数の設定

   プロジェクトルートに`.env.local`ファイルを作成し、バックエンドAPIのベースURLを設定してください。

   ```
   VITE_BASE_URL=http://localhost:3000
   ```

   ※ バックエンドAPIのURLに応じて適宜変更してください。

3. 開発サーバーの起動

   ```bash
   npm run dev
   ```

   起動後、http://localhost:5173 でアクセスできます。

4. テストの実行

   ```bash
   npm run test
   ```

### その他のコマンド

- `npm run lint`: ESLintによるコードチェック
- `npm run format`: Prettierによるコード整形

## その他追加実装など

- Editボタン押下時やエラー発生時に、対象入力欄にフォーカス
- SaveボタンやNew Pageボタンを押下時に、ボタンを非活性にして多重実行を防止
- エラー発生時に対象入力欄にエラーメッセージを表示
- タイトルと本文のそれぞれで入力や保存、キャンセルができるように分離
